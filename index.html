Android 修改 apk 包名

发表于 2019-05-14

工具

  • apktool：用于解包和打包。
  • VSCode 或者其他编辑器：用于关键词的查找和替换。
  • keytool：用于生成签名文件。
  • jarsigner：用于签名 apk。

解包

修改

apktool.yml

packageInfo 节点下增加两个字段：

  • cur_package: xx.yy.zz
  • orig_package: xx.yy.zz

AndroidManifest.xml

  • 根节点中的包名修改 package=”xx.yy.zz”。
  • 生命的 Activity，Service，Provider 等包含包名的，也需要全文替换。

res 目录

  • 如果有些 View 使用了自定义的类型，则需要对包名进行全文替换。

smali

  • 修改包名对应的目录路径
  • smali 中的包名为 Lxx/yy/zz，注意签名的 L

打包

xxx 为解包的目录。
yyy.apk 为重打包的 apk。

签名

生成签名文件

  • cert.keystore：生成的证书文件
  • xxx：证书别名

重新签名

  • cert.keystore 是上一步生成的证书文件
  • yyy-signed.apk 是签名后的 apk
  • yyy.apk 是重新打包的 apk
  • xxx 是证书别名

Unity 单元测试

发表于 2019-05-07

Unity Editor 引入了 NUnit 库来提供单元测试功能，可以通过 Menu->Window->General->Test Runner 打开单元测试面板。

除了传统的 NUnit 的一些测试方法外，Unity 还加入了特有的测试工具类，如 IMonoBehaviourTest，MonoBehaviourTest 等。

基础用法

基础用法和 NUnit 相差不大，如最简单的比较测试。

当然，也可以通过引入 UnityEngine 命名空间，对 Unity 中的类和对象进行测试。

独立的命名空间

通常在简单的项目中，我们定义的类型，通常在 Unity 默认的命名空间下，这样会打包到 Assembly-CSharp.dll 中，这个库是默认被加载的；而如果是独立命名空间的源文件或者库，则需要通过定义 Unity 的 Assembly Definition，并关联到单元测试工程的 Assembly Definition 中，才能够正常使用。
